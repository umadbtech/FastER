Feature Modules
1) Auth & Session

Purpose: Create accounts, authenticate users, manage sessions.

Primary entrypoints

Email signup → /auth/v1/signup + /auth/v1/verify (type=signup)

Email login → /auth/v1/token?grant_type=password

Phone OTP → /auth/v1/verify (type=sms)

Logout → /auth/v1/logout

Password reset → /auth/v1/recover + /auth/v1/verify (type=recovery) + /auth/v1/user

UI surfaces

Signup screen

OTP verification screen

Login screen

Forgot password flow

Logout action

2) Onboarding (Festival-scoped)

Purpose: Collect required profile info to activate membership for a festival.

Primary entrypoints

Prerequisite after login: POST /rest/v1/rpc/ensure_festival_onboarding

Resume & prefill: GET /functions/v1/profile-summary

Write steps:

POST /functions/v1/save-username

POST /functions/v1/save-demographics

POST /functions/v1/save-emergency-contact

POST /functions/v1/accept-terms

Optional profile enhancements

Save legal name → /functions/v1/save-profile-name

Upload avatar → /functions/v1/upload-avatar

Get signed avatar URL → /functions/v1/avatar-url

UI surfaces

7-step onboarding flow (username, demographics, emergency contact, terms…)

Profile edit screens (name/avatar)

Frontend rule

After each save call, use {status, activated, missing[]} to decide next step.

Onboarding complete when status="active" OR missing=[].

3) Home (App Shell Content)

Purpose: Load server-driven Home content and tile ordering.

Primary entrypoint (recommended)

GET /functions/v1/app-home-bundle?festival_slug=<slug>
Supports ETag caching (If-None-Match → 304)

Supporting endpoints

Festival header → GET /functions/v1/festival-header?festival_slug=<slug>

Simple home bundle → GET /functions/v1/content-home?festival_slug=<slug> (legacy/lightweight)

UI surfaces

Home screen hero carousel, announcements, upcoming events

Server-driven tiles (“Festival Experience”, “Lineup & Schedule”, “Safety”, “FAQ”)

4) Lineup & Artist Content

Purpose: Browse artists and view artist detail.

Primary entrypoints

Lineup list → GET /functions/v1/content-lineup?festival_slug=<slug>

Artist detail → GET /functions/v1/content-artist-detail?festival_slug=<slug>&artist_slug=<slug>

UI surfaces

Lineup page (grid/list)

Artist detail page

5) Schedule

Purpose: View event schedule by stages/time.

Primary entrypoint

Stage schedule → GET /functions/v1/content-stage-schedule?festival_slug=<slug>

UI surfaces

Schedule page (day tabs, stage filters, timeline list)

6) Map & Venues

Purpose: Show venues/pins for festival map features.

Primary entrypoint

Map content → GET /functions/v1/content-map?festival_slug=<slug>

UI surfaces

Map screen (venues, stages, POIs depending on data)

7) Festival Experience (Amenities / POIs)

Purpose: Amenities directory (medical, water, restrooms, etc.) used by map filters + lists.

Primary entrypoints

Categories → GET /functions/v1/festival-experience-categories?festival_slug=<slug>

Locations by category → GET /functions/v1/festival-experience-locations?festival_slug=<slug>&category=<key>

Single location detail → GET /functions/v1/festival-experience-location?id=<location_id>

Bundle entrypoints (recommended)

App experience bundle → GET /functions/v1/app-experience-bundle?festival_slug=<slug>

Offline bundle → GET /functions/v1/offline-bundle?festival_slug=<slug>

UI surfaces

Amenities category chips + list

Map pins filtered by category

Location detail sheet

8) Friendships (Social)

Purpose: Find festival members, send/accept requests, manage relationships.

Prerequisite

POST /rest/v1/rpc/ensure_festival_onboarding (after login)

Primary entrypoints

Search members → POST /rest/v1/rpc/search_festival_members

Request → POST /functions/v1/request-friendship

Respond (accept/reject/block/unblock) → POST /functions/v1/respond-friendship

List friendships (REST) → GET /rest/v1/friendships?...

UI surfaces

Search users

Pending requests

Friends list

Profile actions (block/unblock)
